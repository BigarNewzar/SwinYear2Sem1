Assumptions

It is assumed that you do not have "Java - JDK", "Apache NetBeans IDE" and "GlassFish Server" on your machines. We have very specific version requirements to these software. Please see "Software Required" section for details and urls for download.

In case you have installed the relevant software but with a different version (earlier or later), you may need to install the right versions for the Lab work.

Software Required

You will need the following software to complete the tasks in this Lab Exercise

  1. Open JDK 1.8.0 update 282 or later (or Open JDK 8)

    1. GlassFish Server only accepts JDK 8 (either Oracle’s JDK 8 or OpenJDK 8). You will see this later when you start the GlassFish Server with JDK 9 or later.

    2. We use OpenJDK 8 instead of the Oracle JDK due to Oracle’s licensing issues.

    3. Having said that you may want to try using Oracle JDK 1.8.0 update 282 or later for your own personal study. But I have not tried it and hence cannot recommend it.

  2. Apache NetBeans 12.2 (or, NetBeans)

    1. Eclipse GlassFish Server version 5.1.0 Full Platform (or, GlassFish 5.1.0, GlassFish)

      This is bundled with Apache NetBeans 12.2.

    2. Java DB (or Apache Derby)

      This is a database server, also bundled with GlassFish Server.

2. Set up JDK policy for database connection

  1. Open a "File Explorer" (MS Windows) or "Finder" (Mac OSX) window

  2. Navigate to the folder ${JDK_ROOT}\jre\lib\security

    Remember to substitute ${JDK_ROOT} with the actual folder name you recorded earlier

  3. Locate the file java.policy

  4. Open your favourite text editor in administrator mode to edit the java.policy file

    Make sure you have a backup copy of the file for restoration, in case you mess up the original file.

  5. Locate the following line in the java.policy file

     // Standard extensions ...

  6. Add the following lines to the java.policy file just after the line // Standard extensions …​

    grant {
   // add for learning Enterprise Development
   permission java.net.SocketPermission "localhost:1527", "listen,resolve";
 };
    You can add these lines at the end of the file. However, I think it is easier to find these lines at the beginning of a file.

  7. Save the java.policy file

    Remember to take a screen shot of these lines in the "java.policy" file as evidence for your portfolio submission.

3. Install Apache NetBeans IDE version 12.2

We will use Apache NetBeans IDE version 12.2. You need to install JDK first before installing NetBeans.

[MS Windows users] We will perform a customized installation. Please read the instructions below carefully during the installation. Do not accept defaults all the way through, it won’t work!

[Mac OSX users] We will perform a standardized installation. Please accept all defaults. However, we will do some customization afterwards. Please read the instructions below carefully.

  1. During the installation, NetBeans will obtain and record the JDK version that you have in your system.

  2. If you only have one version of JDK, that will be it.

  3. If you have multiple versions of JDK (e.g. JDK8 and JDK11), NetBeans will "self-determine" one of them, most probably the latest version (e.g. JDK11). This will cause some issues in the GlassFish Server. After the installation, you need to follow the steps in Section 4 below to check and make sure things are fine.

  1. Download the Apache NetBeans version 12.2 installer

    Remember the folder where you store the NetBeans installer.

  2. [MS Windows] Install NetBeans

    1. Go to the folder where you stored the downloaded NetBeans installer

    2. Run the installation program as an administrator

      Select it, right click your mouse and select "Run as administrator"

    3. Answer "Yes" when prompted whether you want to run the installer

      NetBeans will extract and open the Apache NetBeans IDE Installer window.

    4. In the Apache NetBeans IDE Installer window, do the following

      1. Check that the following five items are pre-selected for installation

        1. Base IDE

        2. Java SE

        3. Java EE

        4. HTML5/JavaScript

        5. PHP

      2. Click the Next > button

      3. Accept the license agreement after you have a good look at it :-)

      4. Enter C:\apps\NetBeans-12.2 in the Install the Apache NetBeans IDE to text field

        1. If the folder name of NetBeans IDE contains space characters (for example, C:\Program Files\NetBeans-12.2 - the default for MS Windows), NetBeans will not work for our Jakarta EE development work.

        2. A good folder name that works for our Jakarta EE development is C:\apps\NetBeans-12.2 for MS Windows.

        3. It is better to change it now.

      5. Keep the pre-selected JDK in the JDK\TM for the Apache NetBeans IDE text field

      6. Click the Next > button

      7. Click the Install button

      8. Wait for the installation to finish

  3. [(Intel-based) Mac OSX] Install NetBeans

    1. Go to the folder where you stored the downloaded NetBeans installer

    2. Run the installation program as an administrator

    3. Follow the on-screen prompt to install NetBeans using the default options

    4. Wait for the installation to finish

    5. [Customization] Fix the name of the default installation folder on Mac OSX

      1. If the folder name of NetBeans IDE contains space characters (for example, /Applications/NetBeans/Apache NetBeans 12.2 - the default for Mac OSX), NetBeans will not work for our Jakarta EE development work.

      2. A good folder name that works for our Jakarta EE development is /Applications/NetBeans/Apache_NetBeans_12.2 for Mac OSX.

      3. We need to change it now after the installation because we could not do this during the installation.

      1. Open a terminal window

      2. Change directory to /Applications/NetBeans in the terminal window

        You will find Apache NetBeans 12.2.app in the directory. This does not work for us. We need a folder name that does not have space character in between.

      3. Change the directory name to Apache_NetBeans_12.2.app

        Enter sudo mv Apache\ NetBeans\ 12.2.app Apache_NetBeans_12.2.app

4. Check Java SE Platform in Apache NetBeans IDE

GlassFish Server is only compatible with JDK 8. If you have installed JDK 8 and only JDK 8, you can skip this step. In case, you have two different JDK versions (like me :-( - installing JDK 11 before installing JDK 8), you need to follow the steps below to check and make sure the right Java SE platform has been "registered" in NetBeans.

  1. Start NetBeans (if you have not done so)

  2. Select Tools  Java Platforms in the pull down menu

  3. In the Java Platform Manager window, do the following

    1. In the "Platforms:" text area, check whether you have either "JDK 1.8 (Default)" or "JDK 1.8"

      Yes. Although Oracle’s external announcement for Java version 8 is JDK 8, its internal code for Java version 8 is still "JDK 1.8".

      1. If yes, that is fine. Go to here.

      2. If no, do the following

        1. Click the Add Platform…​ button (located just below the "Platforms" text area)

        2. In the "Add Java Platform" window

          1. Select "Java Standard Edition"

          2. Click the Next > button

            NetBeans will display another "Add Java Platform" window with several Java versions (JDK and JRE) for you to select. Make sure you select "JDK".

          4. Click the Next > button

            NetBeans now has "JDK 1.8" in the "Java SE Platforms".

9. Known Restrictions and Issues on your work in COS30041 CSSS

9.1. Restrictions on the folder name of your work, especially for those NetBeans Projects

Before you create your first NetBeans Projects for COS30041, you need to understand the restrictions on the naming of the folder containing the relevant NetBeans Projects.

The path name of the folder containing these NetBeans Projects must contain only alphanumeric characters (i.e. 'a-z', 'A-Z' and '0-9'), underscore character '_' and minus character '-'.

For MS Windows, a good example is C:\Users\wlai\Documents\Swinburne\2022_S1\COS30041\Labs.

For Mac OSX, a good example is /Users/wlai/Documents/Swinburne/2022_S1/COS30041/Labs/.

The colon character ‘:’ next to the drive identifier (e.g. "C:") and the folder separator character (e.g. '\' in MS Windows or '/' in Mac OSX) are exceptions.

Do NOT use any space characters '  ' or any other characters such as % and +. Bad examples are C:\Users\wlai\2021 Semester1\COS30041\NBProjects and C:\Users\wlai\Create%20Soft\NBProjects.
Remember to record the actual folder name, e.g. C:\Users\wlai\COS30041\Labs, because you need to use it in future Labs.

9.2. Issues on using "Auto-Synchronization" software (e.g. One-Drive, Dropbox, Google Drive) and "Virus Scanner" on the folder of your work

Yes, it is good to use these "auto-synch" software to manage your work across different machines. I am using it as well.

Issue 1: These auto-synch software may cause issues when you are doing your COS30041 work, especially saving / storing your Apache NetBeans Projects.

Resolution: Stop the "auto-synch" feature temporarily while you are doing your work will resolve the issues.

Why is that: In a nutshell, these software will put a lock on the files when they try to synchronize these files. During this time, the NetBeans IDE and GlassFish Server will not be able to "do their work (build the jar files, deploy the software on to the GlassFish server and run the software in GlassFish server, …​)" properly. So, the trick is to "pause any synchronization" while you are doing your work, and then "resume the synchronization" afterwards.

Issue 2 [MS Windows only]: MS Windows machines have some "mystery" software (e.g. BitLocker and its anti-virus scanner) that may "lock" the files when they are scanning the files. Mac OSX machines are better in this perspective because the file system of the O/S uses a different read write mechanism. I am not sure about Linux machines, have not tried it on Linux.

Resolution: Stop these software, if you have it running, while you are doing your work.

Why is that: Explain above.
